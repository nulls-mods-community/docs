# UUID и версии модов

### UUID (Modification-Id)

**Для распакованных модов (папки):**
UUID определяется названием папки. Создайте папку с именем, совпадающим с UUID, и поместите файлы мода внутрь:

```
📁 bd850738-67cb-4583-92c4-a12f4f6080d1
└─ 📝 content.json
```

**Для подписанных файлов (.NullsBrawlAssets):**
UUID указывается в `META-INF/MANIFEST.MF` в поле `Modification-Id`.

**Как сгенерировать:**
- Браузер: [uuidgenerator.net](https://www.uuidgenerator.net)
- Linux/macOS: `uuidgen`
- PowerShell: `[guid]::NewGuid()`

При обновлении мода UUID остается прежним. Для нового мода — новый UUID.

> **Примечание:** автоматическая подпись через бота генерирует UUID на основе
> связки `@author` + `@title`, поэтому он будет постоянным для одного мода.

### Версии (@version)

Версия указывается в `content.json` в поле `@version` в формате `X.Y.Z`:

```json
{
  "@title": "Мой мод",
  "@version": "1.2.0"
}
```

Если поле отсутствует, версия по умолчанию — `"1.0.0"`.

Строгое соблюдение SemVer не требуется, главное, чтобы у новой версии числа
(слева направо) были больше, чем у предыдущей:

`1.0.0` → `1.0.1` → `1.1.0` → `2.0.0`

Модлоадер считает более новой ту версию, у которой больше число в старшем разряде.
В библиотеке пользователь видит самую новую версию мода.

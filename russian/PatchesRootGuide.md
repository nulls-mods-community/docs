# Продвинутая работа с патчами (@patches и @root)

`@patches` и `@root` упрощают поддержку больших модов. Работают как в корне мода,
так и внутри отдельных фич.

### @patches

Выносит JSON-объект с патчами в отдельный файл, разгружая `content.json`.

```json
{
  "@title": "Мой мод",
  "@patches": "patches/skins.json"
}
```

`patches/skins.json`:
```json
{
  "skins": { ... },
  "skin_confs": { ... }
}
```

Путь указывается относительно корня архива — `@root` на него не влияет.

> **Важно:** если указан `@patches`, никакие другие патчи в том же объекте
> недопустимы — мод не подпишут.

### @root

Указывает папку с ресурсами мода внутри архива.

```json
{
  "@root": "my_res"
}
```

Игра будет искать файлы относительно `my_res`, а не корня архива.

> Начиная с `@gv:64` работает стабильно. В версии 63.286 мог не работать.

### Совместное использование

```json
{
  "@title": "Большой мод",
  "@root": "assets",
  "@features": {
    "feature1": {
      "@name": "Фича 1",
      "@root": "feature1_res",
      "@patches": "patches/feature1.json"
    }
  }
}
```

Пути в `@patches` всегда относительно корня архива, независимо от `@root`.

# Фичи (@features) и группы фич (@feature_groups)

Фичи позволяют разбить мод на независимые включаемые/выключаемые части.
Пользователь выбирает, какие из них активировать.

### @features

```json
"@features": {
  "feature_id": {
    "@name": "Название фичи",           // обязательно
    "@description": "Описание",         // опционально
    "@enabled": true,                   // включена по умолчанию (по умолчанию: true)
    "@priority": 0,                     // порядок загрузки (меньше = раньше)
    "@conflicts": ["other_feature"],    // несовместимые фичи
    "@root": "папка_с_ресурсами",
    "@patches": "путь/к/файлу.json"
    // любые патчи таблиц, как в корне мода
  }
}
```

**Пример:**
```json
{
  "@title": "Мод с фичами",
  "@features": {
    "extra_skins": {
      "@name": "Дополнительные скины",
      "@enabled": true,
      "skins": {
        "Colt_skin": { "Character": "Colt", "Model": "colt_gold.scw" }
      }
    },
    "custom_music": {
      "@name": "Другая музыка",
      "@enabled": false,
      "music": { "lobby": "mymusic.mp3" }
    }
  }
}
```

### @feature_groups

Объединяют фичи в визуальный блок. Типы:
- `DEFAULT` — чекбоксы, можно включить несколько.
- `RADIO_GROUP` — радиокнопки, можно включить только одну.

```json
"@feature_groups": {
  "group_id": {
    "@name": "Название группы",
    "@description": "Описание",
    "@type": "RADIO_GROUP",
    "@features": ["feature_id1", "feature_id2"]
  }
}
```

> **Важно для RADIO_GROUP:** по умолчанию должна быть включена не более одной
> фичи из группы — иначе мод не подпишут.

**Пример:**
```json
{
  "@title": "Выбор режима",
  "@features": {
    "mode_easy": { "@name": "Лёгкий режим", "@enabled": true },
    "mode_hard": { "@name": "Сложный режим", "@enabled": false }
  },
  "@feature_groups": {
    "difficulty": {
      "@name": "Сложность",
      "@type": "RADIO_GROUP",
      "@features": ["mode_easy", "mode_hard"]
    }
  }
}
```

### Порядок загрузки

1. Корневые патчи `content.json` всегда активны.
2. Включенные фичи загружаются по `@priority` (меньше значит раньше), при равных приоритетах сортируются лексикографически по ID.

Более поздняя фича перекрывает данные предыдущей.

### Ограничения

Нельзя одновременно указывать `@patches` и вложенные патчи в теле фичи — только что-то одно.

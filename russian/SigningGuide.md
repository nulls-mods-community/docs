# Подпись модов — автоматическая и ручная (Pro)

### Автоматическая подпись (бот @nulls_mods_bot)

Подходит для простых модов из одного JSON-файла без ресурсов.

1. Отправьте `.json` боту [@nulls_mods_bot](https://t.me/nulls_mods_bot) или
   в топик [#Signing Requests Lite](https://t.me/nb_mods/4827) командой `/sign`.
2. Бот вернёт подписанный `.NullsBrawlAssets`.
**Ограничения:**
- Только `content.json` — без текстур, звуков, иконок.
- Срок действия подписанного файла — **3 дня**.
- `@author` заполняется автоматически ником и Telegram ID.
- UUID генерируется из связки `@author` + `@title`.
- `icon.png` не поддерживается.

### Ручная подпись (Pro)

Подходит для любых модов с ресурсами и для публикации в библиотеке.

**Подготовка ZIP-архива:**
- `content.json` в корне архива (обязательно)
- файлы ресурсов в соответствующих папках (`sc3d`, `sfx`, `music` и т.д.)
- `icon.png` — квадратная PNG, не более 640×640 (опционально)

**Процесс:**
1. Сгенерируйте UUID (см. гайд по UUID). Для нового мода — новый UUID,
   для обновления — тот же.
2. Отправьте архив в топик **#Signing Requests Pro**.
3. В сообщении укажите: название, описание, UUID, версию (для обновлений),
   ссылки на скриншоты/видео из @nb_mods (если есть).
4. Дождитесь проверки — от нескольких часов до нескольких дней.

**Требования:**
- Название и описание без ошибок, желательно с локализацией на RU и EN.
- Чужой UUID без разрешения автора запрещён.
- При обновлении версия должна быть больше предыдущей.

**Публикация в библиотеку:**
Добавьте в сообщение «Добавьте в библиотеку, пожалуйста». Мод должен вносить
существенные изменения.

### Перед отправкой

- Убедитесь, что `content.json` валидный.
- Все пути к файлам указаны верно.
- Для модов с `RADIO_GROUP` — включена не более одной фичи по умолчанию.
